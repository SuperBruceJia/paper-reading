    深度学习模型中的卷积计算分析及优化方法总结（下）

    | 导语 本文是深度学习模型中的卷积计算分析及优化方法总结的第二篇文章，总结自AlexNet以来出现的各种模型层面的卷积优化方法，并简要介绍一些未来的卷积优化研究方向。

    模型层面的优化

    通过上一篇文章，我们已经了解两件事：
    第一，最底层的卷积是如何实现的。
    第二，不同类型的卷积操作对内存、计算以及精度的影响。
    这一篇文章将会探讨如何将不同的卷积以不同的形式嵌入到模型结构中。

    开始一切的模型

    从很多方面来看，AlexNet的模型结构是对于98年出现的LeNet的致敬和扩展。不过，除了沿用了卷积、池化等操作之外，AlexNet第一次引入了分组卷积（在双GPU上分为两组）、ReLU、Dropout、LRN等几个重要概念（当然还包括数据增强，这里只提到跟网络结构有关的最重要的几点），让深度学习方法在大规模图像多分类场景下真正开始得到关注。也最终开始了本世纪的新一轮AI革命浪潮。
    AlexNet这种以卷积和池化层叠来获取特征表示，以几层全连接和softmax进行最终的多分类结构深刻的影响了之后的几乎所有CNN模型架构。


    [ AlexNet可以分为特征提取模块和全连接模块两部分 ]

    为了在AlexNet的基础上进一步提高模型的性能，研究人员在接下来的几年里分别从网络模型和卷积的不同维度进行了扩展和探索。这里可以把不同的方向归纳为以下四点，最后我们总结EfficientNets中对其他方法中几个维度变换的集成，一个统一的对卷积和网络结构的缩放方法。也简单分析这一方法跟网络结构搜索(Network Architecture Search)的联系。

    子模块堆叠法

    VGG模型尝试增加卷积网络的层数。为了保证层数增加的情况下参数大小不致过大，采用了能够包含上下左右和中心这几个基本特征的最小可能的kernel尺寸：3x3。这个选择带来的额外的好处是，不同层数的3x3卷积叠加之后，可以制造与5x5，7x7等相同大小的接受域。另外，参数大小也更小了。


    [ 不同的VGG网络模型架构图 ]

    从VGG模型的结构可以看出，它沿用了AlexNet的先采用多层卷积进行特征提取，再采用多层全连接和softmax进行最终的分类这个设计思路。而：把相同卷积操作组成的基础子模块堆叠起来，是它给CNN模型设计带来的主要贡献。
    同样的思路，将基础子模块换成depthwise separable convolution，就得到了mobilenets的基本架构。

    [ mobilenetv1结构图。红框中为模型中的两种depthwise separable convolution。 ]

    残差模块

    残差模块的初衷是为了让CNN模型层数达到更深。LSTM的创始人JS和他的团队在Highway Network里第一次提到了使用可学习的类似门电路结构决定是否让某些路径忽略模型的一些层。这几乎和残差模块的shortcut设计很接近。然而Kaiming He的团队的shortcut层没有参数，表达了真正的残差而不是shortcut和模型某个局部输出的线性叠加。在理论上和实际上都取得了比Highway Network更好的结果。在最初的Residual Network上获得好的效果之后，残差模块被进一步应用到几乎所有之后出现的CNN模型上。包括DenseNet，U-Net，以及序列模型中的WaveNet，NMT，以及Transformer上。可以说已经成为了DNN模型的基础模块。


    [ 一个残差模块 ]

    残差模块本身的实现非常容易，只需要在多层卷积和非线性激活层之后添加一路shortcut，将原本的输入和多层卷积的输出进行相加即可。其高层架构仍然沿用了多层卷积用于特征提取，全连接和softmax用于多分类这一准则。
    另外值得一提的是bottleneck模块，即使用1x1卷积对输出的通道数量进行增减。这一改进使得更深的模型可以在可负担的时间内进行。事实上ResNet-50/152等较为流行的残差模型都是采用了bottleneck模块的。

    [ 标准残差模块（左）与bottleneck模块（右）对比图 ]

    切分-变换-融合法

    从VGG到残差模块，CNN模型在链式结构的基础上不断变深。
    与此同时，另一个维度的探索在以谷歌大脑（以及作为主要完成人的几十个实习生们 :-P）为代表的研究团队中展开。所谓切分-变换-融合，即split-transform-merge。


    [ Inception初版GoogLeNet的基本模块 ]

    这一想法的出发点是认为通道间以及空间之间的相互关联是可以被分开探索的（分别采用1x1卷积以及不同kernel尺寸的多路卷积），将这些变换结果再融合起来就能得到更丰富的特征空间表达。同时，这里是采用1x1卷积（以及各类池化）进行维度降低的最初发源地。
    最后，有必要提到的是激发这类切分-变换-融合结构的最早的工作，那就是Network In Network (NIN)。这篇文章对卷积进行了细致的观察，发现卷积的receptive field和filter之间的操作是一个简单的线性变换（内积），于是提出将这一线性过程改变为非线性过程的方法：

    [ 普通的卷积（左）和添加了MLP的卷积（右侧） ]

    值得一提的是，在多层卷积结束之后，将最后一层的所有kernel的输出结果通过全局的空间取均值化为一维向量也是本文第一次提出，并在其后被所有CNN采用的重要优化。因为均值这种操作使得反向传播时，分类信息和特征信息之间有了显式的关联。

    [ 一个包含多层MLPConv和最终的全局Avg Pooling的NIN结构 ]

    复杂子图堆叠法

    Inception出了很多版本（初版和v3是两篇值得细读的文章在我看来），回想那个Pre-AutoML时代，谷歌的很多实习生都在历次Inception版本上无谓的耗费了大量脑力和时间。此时，Kaiming He的团队再一次提出了新的想法，或一个此类方法的总结：即采用某个特定的拓扑结构组织不同空间、通道变换的多路卷积操作，并将此作为重复的基础模块堆叠。ResNeXt重新引入了分组卷积的概念，强调采用同样的拓扑结构的重要性。


    [ 如何用分组卷积实现ResNeXt基本模块 ]

    这里列举的像当初对待特征工程一样，研究人员几乎穷尽了所有智力。当投入的精力和提高的性能呈现递减的回馈时，人们需要另一种方法，一种理想的方法。

    朝一个理想的方向迈进（搜索网络结构）

    经过之前的内容，我相信如果读者朋友具备同样的背景知识又身处谷歌大脑，手握大量TPU计算资源，也会有想出EfficientNet这一算法的可能性。下面我们来介绍EfficientNet。

    这篇文章本质上是这样的思路：根据之前的经验，提高模型性能最直接的办法就是扩展模型的尺度。那么如何扩展？这篇文章把可扩展的维度做了划分，包括：每层卷积的通道数（文章中用宽度来表示）, 模型中的卷积层数, 以及Feature Map（多层卷积之间传递的数据）的空间尺寸。


    [ 对模型结构在不同维度上的扩展方式 ]

    当然，上图(e)这样的组合式扩展模型结构的方法也曾经被人尝试过，但是都是通过手调，本文提出了一种普适的模型结构扩展方法。考虑到卷积操作跟通道数的平方、feature map尺寸的平方以及卷积层数呈正相关，如何在保证计算资源稳定增长的情况下最优的组织不同维度上扩展的强度是这篇文章最大的贡献。

    其中不同维度内扩张的规则如下：

    [ 如何在不同维度上扩张模型规模 ]

    在基线上搜索合适的alpha, beta, 以及gamma取值，得到最优精度的过程可以用一个最优化过程表示。

    [ 最佳模型结构作为一个最优化问题来表述 ]

    在此，（险恶的猜测）作者为了保留更多今后发表续作的机会，仅仅尝试了网格搜索。具体过程分为两步：
    固定phi值为1，根据最优化公式限制网格搜索出最佳的取值：alpha=1.2, beta=1.1, gamma=1.15。
    固定alpha beta以及gamma，根据扩张规则公式，线性提高phi，从而得到一系列EfficientNets。
    实验结果非常好，在此不赘述，感兴趣的读者朋友可以移步论文。

    [ EfficientNets实验结果，蓝框标注应该为此文发表时SOTA的ImageNet上的深度模型识别精度 ]

    在此我仅对可能在其上继续的工作进行一点讨论：
    第一，上述三种维度是否完全覆盖了卷积的表达？是否还可以增加维度？比如ResNeXt分组卷积的这个维度其实是可以用在这里的。
    第二，搜索问题是否可以用网格搜索之外的更智能的算法？

    可以看到这里已经有了NAS的初步结构：定义合适的搜索空间，定义合适的优化方法。网络结构搜索超出了卷积优化的范围，所以在此不表。会专门用一文详述，在优化方法和搜索空间上比单纯的卷积搜索要更加复杂。

    至此，笔者通过两篇文章大体梳理了深度学习中关于卷积的优化问题，希望对鹅厂内从事相关模型优化的同事起到一定的帮助作用。
